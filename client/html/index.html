Introduction

Your Cup journey begins here!

Cup is a free and open source project consisting of a compiler, build system and a package manager. It brings back the simple C-style memory management while having a modern and expressive syntax.

This tutorial should teach you basics of Cup and how to use it. If you encounter any problems or have any questions you can reach out on our Discord Server in the #begginers channel.

Getting Started

Cup is conveniently distributed as a single executable file which means it's portable in it's nature. Cup still can be installed using the cup self install command. You will learn more about commands in the next lesson.

To be able to use Cup you will also need a C compiler of some kind. You can use either GCC, Clang or MSVC. Refer to your platform’s documentation on how to install a given compiler.

Check the installation

To check whether you have installed Cup and a C compiler correctly, open a shell and run cup version. You should see something similar to this:

XXXXXXXXXXXX
XXXXXXXXXXXX
XXXXXXXXXXXX

Troubleshooting

If you see the No C compiler detected! error it means you did something wrong installing the C compiler. You can try a different compiler or ask for help on our Discord Server in the #begginers channel.

Commands

There is a number of commands that are available to you when using Cup. They are designed to automate things and make managing your project easier.

To see this list you can run cup help. To see more info about a given command run cup help followed by the name of a command, for example cup help build.

Complilation commands

  • cup run - Compiles and runs the current package
  • cup build - Compile the current package
  • cup check - Analyzes the current package

Package commands

  • cup new [PACKAGE] - Creates a new package with given name
  • cup update [PACKAGE] - Updates given dependency
  • cup add [PACKAGE] - Adds given dependency
  • cup remove [PACKAGE] - Removes given dependency

Generation commands

  • cup gen docs - Generate documentation for the current package
  • cup gen binds [HEADER] - Generate bindings for a given C header file

Self commands

  • cup self install - Install the Cup Toolkit
  • cup self update - Update the Cup Toolkit
  • cup self uninstall - Uninstall the Cup Toolkit

First Program

Now that everything is installed and you know how to use commands, let’s write your first Cup program. It’s traditional when learning a new language to write a little program that shows the text Hello, world! to the screen, so we’ll do just this here!

Creating a new project

Creating a empty project with Cup is very simple. All you need to do is run cup new followed by the name of the project, for example cup new hello-world. This will create a folder with a name hello-world and a file called main.cp inside it.

Writing and running a Cup Program

Now open the main.cp file that just got created in the editor of your choice and type in this code:

sub main() {
    std:fmt:print("Hello, world!");
};

Save the file and run cup run. This will compile and run the code for you. You should see this on your screen:

Hello, world!

If you can see that on your screen, congratulations! You're officialy a Cup programmer!

What just happened?

Let’s review in detail what just happened in your “Hello, world!” program. Here’s the first piece of the puzzle:

sub main() {
    std:fmt:print("Hello, world!");
};

Guessing Game

Now let's up the challenge a little bit. This chapter introduces you to a few common Cup concepts by showing you how to use them in a real program. You’ll learn about var, match, subroutines, using namespaces, and more! The following chapters will explore these ideas in more detail. In this chapter, you’ll practice the fundamentals.

We’ll implement a classic beginner programming problem: a guessing game. Here’s how it works: the program will generate a random integer between 1 and 100. It will then prompt the player to enter a guess. After a guess is entered, the program will indicate whether the guess is too low or too high. If the guess is correct, the game will print a congratulatory message and exit.
